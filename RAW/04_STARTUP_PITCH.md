# 수업의 목표

&gt; 목표: 아이디어를 '투자 가능한 가설'이자 '설명 가능한 사업'으로 번역함.

# 이번 주 수업이 지향하는 3가지 전환

![img-3.jpeg](img-3.jpeg)

## 1. 사고의 전환

기능 관점 → 투자 가능성 관점

학생들이 사용자나 기능 관점에서 벗어나 시장 성, 수익성, 확장성의 관점으로 구조적으로 생 각하도록 만들.

![img-4.jpeg](img-4.jpeg)

## 2. 표현의 전환

기능 나열 → 사업의 언어

"AI로 추천해주는 앱임"이 아니라 문제의 강도, 시장 규모, 성장 구조를 설명하는 '사업의 언 어'로 말하게 함.

![img-5.jpeg](img-5.jpeg)

## 3. 실행의 전환

발표 목적 → 허점 발견 및 가설 도출

단순히 발표를 잘하는 것이 목적이 아님. 피치 를 통해 스스로의 논리적 빈틈을 찾고 검증해 야 할 가설을 식별함.

왜 하필 '투자자 관점'인가?

돈을 받기 위해서? (X)

기초 수업일수록 투자자 관점의 훈련이 일찍 들어가야 함.
이는 단순히 자금 조달의 문제가 아니라, 사업을 가장 압축적이고 냉정
하게 구조화하는 훈련이기 때문임.

투자자의 근본적인 질문 = 창업자의 필수 질문

* 이 문제는 충분히 크고 고통스러운가?
* 이 해결책은 기존 대안보다 정말 압도적인가?
* 이 시장은 폭발적으로 커질 수 있는가?
* 이 비즈니스 모델은 확장 가능한가?

# 흔히 범하는 2가지 오류

초기 창업자들이 피치에서 가장 흔하게 실패하는 이유

## 1. 기능 과잉 설명

자기 서비스가 얼마나 '다양한 기능'을 가졌는지 나열하는 데 급급함.

- 기능이 많다고 결코 좋은 사업이 아님.
- 오히려 단 하나의 날카로운 '핵심 가치(Core Value)'가 흐려짐.
- 투자자는 기능 명세서가 아닌 비즈니스 작동 구조를 듣고 싶어 함.

## 2. 문제의 모호함

"요즘 사람들이 바쁘다", "AI가 대세다"처럼 너무 넓고 흐릿한 현상을 문제로 포장함.

- 두루뭉술한 문제 설정으로는 타겟 고객이 좁혀지지 않음.
- 고객이 기꺼이 지갑을 열 만큼의 구체적인 고통(Pain)이 보이지 않는 피치는 실패함.

# 투자자가 보는 5가지 핵심 비즈니스 요소

투자자는 아이디어의 귀여움을 보는 것이 아니라, 아래의 '구조'를 봄.

1. 문제의 강도: "있으면 좋은가?" vs "없으면 고통스러운가?" (비타민인가, 진통제인가)
2. 타겟의 명확성: "모두를 위한 서비스"는 대개 "아무도 쓰지 않는 서비스"임. 최초의 고객은 누구인가?
3. 해결책의 차별성: 기존 대안 대비 차별성이 단순한 UX/기능 편의성인가, 아니면 가치사슬을 흔드는 구조적 혁신인가?
4. 시장의 크기: 작고 예쁜 틈새시장인가, 아니면 초기엔 불완전해 보여도 폭발적으로 커질 수 있는 거대 시장(TAM)인가?
5. 성장 기능성: 열심히 영업하면 키지는 '선형 성장' 구조인가, 아니면 S/W의 네트워크 효과를 탄 '지수적 성장' 구조인가?

# 설명 가능한 사업: 5문장 피치 구조

설명 순서는 절대 '서비스 소개'가 먼저가 아님.

1. 어떤 타겟 고객이 어떤 '문제'를 겪고 있음.
2. 현재 그들의 '기존 대만'은 ~한계 때문에 충분하지 않음.
3. 우리는 어떤 '해결책'을 제안함.
4. 우리의 핵심 '차별점(구조)'은 무엇임.
5. 그래서 이 시장에서 어떻게 폭발적으로 '성장'할 수 있음.

![img-8.jpeg](img-8.jpeg)

‘설명 가능한 사업’으로 압축하라

![img-9.jpeg](img-9.jpeg)

좋은 사업은 짧은 시간 안에 논리적으로 설명 가능해야 합니다.

1. 누구에게 (Target)
2. 어떤 문제가 있고 (Problem)
3. 어떤 방식으로 해결하며 (Solution)
4. 기존 대비 왜 다르고 (Differentiation)
5. 어떻게 성장할 수 있는가 (Growth)
